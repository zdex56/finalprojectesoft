## описание проекта

stacy — это веб преложение планировщик задач, который первым делом будет давать людям то зачем они зашли на сайт, их задачи напоминания никакого всплывающего мусора рекламы, при этом очень часто делать окцент на то что всё будет оч просто по визуалу, и понятно любому, ( очень хочу сделать адаптивность под мобилки если будет время)

**для кого:** студенты и работники, которым нужен простой планировщик.

**что решает:** убирает отвлекающий мусор из существующих планировщиков, задачи можно создавать, делиться и привязывать к месту на карте.

## user stories

1. как пользователь, я хочу зарегистрироваться и войти.
2. как пользователь, я хочу создать задачу с местом на карте.
3. как пользователь, я хочу добавить другого пользователя в задачу.
4. как пользователь, я хочу изменить статус задачи (новая/в работе/готова).
5. как пользователь, я хочу фильтровать задачи по статусу.
6. как администратор, я хочу видеть на карте все задачи всех пользователей, чтобы контролировать общую картину.

## сущности (база данных)

- **user** — пользователь (id, email, пароль, роль)
- **task** — задача (id, название, статус, координаты, создатель)
- **taskassignee** — связка (кто в какой задаче исполнитель)

( явно бд я улучшу со временем, мне очень тяжело думать о логике бд до этапа её разработки, явно будут изменения в будущем, щас краткая база)
связи: user → task (1 ко многим, как создатель), user ↔ task (многие ко многим через taskassignee, как исполнители)

## экраны

1. **логин/регистрация** — вход и создание аккаунта
2. **дашборд** — список моих задач + фильтр + пагинация
3. **карточка задачи** — детали + карта с маркером + кнопка смены статуса
4. **создание задачи** — форма + карта для выбора места
5. **личный кабинет** — смена пароля
6. **админ-панель** — поиск пользователя + просмотр всех его задач на карте и списком
